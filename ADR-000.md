# ADR-000: Architecture Decisions for TCHAK FREE V0

## Statut
Accepté

## Décisions

### 1. Framework Desktop : Electron
- **Choix** : Electron avec React.
- **Justification** : Permet l'accès au système de fichiers local (`fs`), la gestion de la chaîne de confiance et l'export PDF tout en réutilisant les compétences web.

### 2. Distribution : Netlify Functions (ZIP Gen)
- **Choix** : Les données du formulaire landing sont traitées par une fonction serverless qui injecte la config (`artist_id`, `seed`) dans un template ZIP.
- **Justification** : Simplifie radicalement l'onboarding ; l'artiste reçoit un binaire "déjà configuré" pour lui.

### 3. Système de Confiance : Chaîne SHA256 Locale
- **Choix** : Chaque certificat est lié au précédent via `SHA256(lastId + timestamp)`.
- **Justification** : Absence de HMAC/Cloud pour rester 100% offline tout en assurant une traçabilité déterministe.

### 4. Limitation : Quota Local 10/mois
- **Choix** : Blocage de l'UI si plus de 10 certificats sont détectés pour le mois calendaire courant.
- **Justification** : Modèle économique "FREE" protégé sans authentification cloud constante.

### 5. Signature : Canvas React
- **Choix** : Signature dessinée sur canvas -> export PNG -> inclusion dans le ZIP template -> insertion PDF.
- **Justification** : Approche low-tech mais efficace pour l'authenticité visuelle.

### 6. Backend Onboarding : Supabase
- **Choix** : Utilisé uniquement pour la landing page (segmentation, analytics).
- **Justification** : L'application locale n'a aucune dépendance runtime avec Supabase.

### 7. Mise à jour : Remplacement & Migration
- **Choix** : Notification de version -> Téléchargement ZIP -> Migration manuelle ou auto du dossier `data/`.
- **Justification** : Évite la complexité de serveurs d'auto-update (SaaS) pour un produit gratuit.
